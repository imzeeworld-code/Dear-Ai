Base Endpoint

All WESEA API requests go to this single endpoint. No complex setup — just a URL, your key, and a query.

GET https://wesea-api.workers.dev/search

Parameters

ParameterTypeDescription
qstringYour search query. Example: yo maps 2026 album
keystringYour WESEA API key from the Keys tab
numnumberResults to return. Default 5, max 10

Integration Code

Paste these two constants into any AI assistant HTML file you build. Your AI will call WESEA automatically when it needs live information.

JavaScript 
// ── WESEA WEB SEARCH ──────────────────────
const WESEA_ENDPOINT = "https://wesea-api.workers.dev/search";
const WESEA_KEY = "YOUR_API_KEY_HERE";

// Call this before asking AI a question
async function weseaSearch(query) {
  const url = `${WESEA_ENDPOINT}?q=${encodeURIComponent(query)}&key=${WESEA_KEY}&num=5`;
  const res  = await fetch(url);
  const data = await res.json();
  // data.results → [{title, snippet, url}, ...]
  return data.results || [];
}

// Feed results to your AI as context
async function askWithSearch(userMessage) {
  const results = await weseaSearch(userMessage);
  const context = results.map((r, i) =>
    `[${i+1}] ${r.title}\n${r.snippet}\nSource: ${r.url}`
  ).join("\n\n");

  // Now pass context + userMessage to your AI
  return `Based on this live data:\n${context}\n\nQuestion: ${userMessage}`;
}
